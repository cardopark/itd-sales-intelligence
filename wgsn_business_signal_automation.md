# WGSN Business Signal Automation

## 목적

고객사/해지 고객사/추천 후보 기업을 대상으로 매월 1회 공개 웹과 공식 재무자료를 확인해 WGSN 세일즈 기회를 자동으로 찾아냅니다.

수동으로 업로드하는 데이터:

- 기존 고객 리스트
- 해지 고객 리스트
- 마케팅 리드풀/리멤버 명함풀

자동으로 업데이트하는 데이터:

- 뉴스/PR 기반 비즈니스 신호
- 공시/재무제표 기반 성장 신호
- 사업보고서 기반 신규 사업/R&D/해외 진출 신호
- 기업별 세일즈 액션 추천

## 월간 실행 흐름

1. 대상 기업을 확정합니다.
   - 현재 고객, 해지 고객, 신규 추천 후보를 합칩니다.
   - 기업명, 브랜드명, 영문명, 법인명, 도메인을 정리합니다.

2. 웹 뉴스와 PR을 검색합니다.
   - 최근 30~45일 기준으로 검색합니다.
   - 신제품, 흑자전환, 매출 증가, 투자 유치, 해외 진출, 유통 확대, 리뉴얼, R&D 채용 키워드를 조합합니다.

3. 재무자료를 확인합니다.
   - 국내 상장사는 OpenDART API로 공시 목록, 사업보고서, 재무제표, 주요 재무지표를 확인합니다.
   - 해외 상장사는 SEC EDGAR, EDINET 등 국가별 공시 데이터로 확장합니다.
   - 비상장사는 공개 뉴스, 투자 보도, 채용, PR 중심으로 판단합니다.

4. 긍정 신호를 분류합니다.
   - 단순 언급, 광고성 기사, 중복 기사, 부정 이슈는 제외합니다.
   - WGSN 구매 가능성과 연결되는 신호만 남깁니다.

5. 세일즈 액션을 생성합니다.
   - 리드풀/명함풀 담당자와 매칭합니다.
   - 담당자가 있으면 이메일/전화 액션을 추천합니다.
   - 담당자가 없으면 탐색 대상 부서와 직함을 추천합니다.

6. 세일즈 담당자에게 월간 이메일을 발송합니다.
   - 담당자별 추천 기업만 묶어서 보냅니다.
   - 세일즈 리더에게는 전체 요약과 담당자별 액션 현황을 보냅니다.
   - 근거 링크와 메시지 초안을 포함합니다.

## 추천 API

### 뉴스/웹 검색

- NAVER Search API: 한국 뉴스 검색에 적합합니다.
- Google Programmable Search JSON API: 글로벌 웹/뉴스룸/PR 페이지 탐색에 활용합니다.
- 기업 공식 RSS/뉴스룸: 주요 고객사는 개별 URL을 등록해 정확도를 높입니다.

### 국내 공시/재무

- OpenDART API: 금융감독원 전자공시시스템의 공시, 사업보고서, 재무제표 데이터를 확인합니다.

우선 연동 항목:

- 회사 고유번호
- 공시 목록
- 단일회사 주요계정
- 단일회사 전체 재무제표
- 단일회사 주요 재무지표
- 사업보고서 원문

### 해외 공시/재무

- SEC EDGAR: 미국 상장사
- EDINET: 일본 상장사
- 유료 데이터 벤더: 글로벌 비상장사와 정교한 재무/투자 데이터가 필요할 때 검토

## 필요한 설정값

API 키는 문서나 화면에 직접 넣지 않고, 별도 환경 설정으로 관리합니다.

- NAVER_CLIENT_ID
- NAVER_CLIENT_SECRET
- GOOGLE_SEARCH_API_KEY
- GOOGLE_SEARCH_ENGINE_ID
- OPENDART_API_KEY
- SEC_USER_AGENT

## 신호 점수화

### 강한 긍정 신호

- 흑자전환
- 영업이익 증가
- 매출 연속 증가
- 대규모 투자 유치
- 해외 시장 본격 진출
- 신규 사업부 또는 신규 브랜드 론칭

추천 액션:

- 세일즈 매니저 직접 컨택
- 과거 사용 부서 재접촉
- 산업별 레퍼런스와 예산 타이밍 중심 메시지

### 중간 긍정 신호

- 신제품 출시
- 유통 채널 확대
- 팝업/콜라보 성과
- 브랜드 리뉴얼
- 상품기획/디자인/R&D 채용 증가

추천 액션:

- 마케팅 리드 후속
- 웨비나 초대
- 샘플 리포트 기반 미팅 제안

### 약한 긍정 신호

- 캠페인 진행
- 단발성 기사
- 임원 인터뷰
- 일반 채용

추천 액션:

- 모니터링 유지
- 다음 달 재확인

## 결과물 형식

월간 자동 실행 후 아래 결과를 생성합니다.

- `signals_auto_YYYY_MM.csv`
- `winback_opportunities_YYYY_MM.csv`
- `target_accounts_YYYY_MM.csv`
- `sales_actions_YYYY_MM.csv`
- `sales_email_digest_YYYY_MM.html`
- 세일즈 매니저별 요약 리포트

## 이메일 발송 설계

### 수신자 설정

수신자 목록은 별도 CSV 또는 설정 파일로 관리합니다.

- account_owner
- email
- role
- receives_individual_digest
- receives_team_summary
- cc_email

역할 예시:

- sales_manager
- sales_leader
- marketing
- admin

### 발송 규칙

- 세일즈 매니저는 본인 담당 기업만 받습니다.
- 세일즈 리더는 전체 기업과 담당자별 액션 수를 받습니다.
- 마케팅 담당자는 마케팅 리드풀에 담당자 정보가 있는 기업만 받습니다.
- 추천 기업이 없으면 "이번 달 신규 추천 없음" 알림을 보낼지 여부를 설정으로 선택합니다.

### 이메일 구성

제목:

- `[WGSN Sales Intelligence] 2026년 5월 추천 기업 8개`
- `[WGSN Win-back] 이번 달 재구독 기회 3개`

본문:

- 요약 지표
- 이번 달 1순위 기업
- 해지 고객 재구독 기회
- 신규 유사 기업 추천
- 담당자 연락처 매칭
- 추천 메시지 초안
- 근거 링크

### 발송 연동

MVP에서는 아래 중 하나를 선택합니다.

- SMTP: 회사 메일 서버 사용
- Gmail API: Google Workspace 사용 시 적합
- Outlook/Microsoft Graph: Microsoft 365 사용 시 적합

필요 설정값:

- EMAIL_PROVIDER
- EMAIL_FROM
- SMTP_HOST
- SMTP_PORT
- SMTP_USERNAME
- SMTP_PASSWORD
- GOOGLE_WORKSPACE_CREDENTIALS 또는 MICROSOFT_GRAPH_CREDENTIALS

보안 원칙:

- 이메일 비밀번호와 API 키는 문서에 저장하지 않습니다.
- 환경 설정 또는 보안 저장소에만 저장합니다.
- 담당자 연락처가 포함된 이메일은 회사 계정 수신자에게만 보냅니다.
- 외부 수신자 발송은 기본적으로 차단합니다.

## 세일즈 메시지 생성 기준

좋은 메시지는 기업 상황과 WGSN의 가치가 바로 이어져야 합니다.

예시:

- 신제품 출시: "최근 신규 라인 출시 이후 다음 시즌 컬러/소재 방향성과 소비자 반응을 함께 점검해볼 수 있습니다."
- 흑자전환: "사업이 다시 확장 국면에 들어간 만큼, 상품기획/R&D 단계에서 트렌드 리서치 활용을 재검토하기 좋은 시점입니다."
- 해외 진출: "해외 시장별 소비자 취향과 리테일 트렌드 차이를 WGSN 데이터로 검토해볼 수 있습니다."
- R&D 채용 증가: "연구개발과 상품기획 조직이 확대되는 시점에는 리서치 기반 의사결정 수요가 커질 가능성이 높습니다."

## 운영상 주의점

- 자동 수집 결과는 세일즈 추천을 위한 보조 판단입니다.
- 기사 원문과 공시 출처 URL을 반드시 함께 저장합니다.
- 개인 연락처는 내부 리드풀/명함풀의 접근 권한 정책을 따라야 합니다.
- 유료 매체나 비공개 자료는 크롤링하지 않고, 허용된 API 또는 라이선스 범위 내에서만 사용합니다.
- 재무제표는 회계 기준과 연결/별도 기준 차이를 표시해야 합니다.
